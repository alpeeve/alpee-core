# alpee-core

Ядро модульной системы управления сервером  **Alpee**.

Проект написано на Rust, запускает модули в виде отдельных процессов, общается с ними по Unix-сокетам (IPC), а фронтенду отдаёт всё это добро через REST и WebSocket.

## Как всё устроено
1. **Ядро (`alpee-core`)** сканирует папку `./modules`.
2. Находит там модули, читает их `module.toml` и запускает релизные бинарники.
3. Модули коннектятся к сокету ядра (`/tmp/alpee-core.sock`) и регистрируют свои фичи (capabilities) и декларативный UI.
4. Фронтенд общается с ядром:
   - Через **REST** для получения списка модулей и отправки синхронных команд.
   - Через **WebSocket** (`/api/ws`) для подписки на реалтайм-метрики/события модулей и асинхронного обмена данными.

## Как написать свой модуль

Клади новый модуль в `./modules/<имя_модуля>/`. Внутри обязательно должны быть:
1. `module.toml` — конфиг с метаданными.
2. Бинарник в папке `target/release/` (имя бинарника должно совпадать с полем `binary` в TOML или с именем модуля).

При запуске ядро передает модулю две переменные окружения:
* `ALPEE_SOCKET` — путь к Unix-сокету ядра.
* `ALPEE_MODULE_NAME` — имя модуля.

### Протокол общения (IPC)
Обмен сообщениями идет фреймами через `LengthDelimitedCodec` (библиотека `tokio-util`), данные сериализуются в JSON.

Сразу после подключения модуль обязан отправить сообщение регистрации:
```json
{
  "Register": {
    "name": "my-module",
    "version": "0.1.0",
    "capabilities": ["metrics", "terminal"],
    "ui_components": [
      { "name": "cpu_chart", "component_type": "chart", "endpoint": "/ui/cpu" }
    ]
  }
}
```

Дальше модуль периодически шлет `Heartbeat` и шлет события (`Event`), а также слушает команды (`Command`) от ядра и отвечает на них (`Response`), связывая их по UUID `id`.
